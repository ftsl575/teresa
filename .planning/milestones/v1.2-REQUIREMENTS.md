# Requirements: Teresa — v1.2 Output structure reorganization

**Defined:** 2026-06-07
**Core Value:** The classifier produces correct, deterministic, audited artifacts for every supported vendor. Everything else is plumbing.

**Milestone scope:** Routing-only. Move every output artifact into three purpose-named buckets under `output_root`, keyed by `<vendor>/<spec>/`, with **zero content changes**. The only content-adjacent change is renaming the `branded` workbook to a Russian filename on its move into READY (filename, not bytes).

---

## v1.2 Requirements

### Bucket Layout (LAYOUT)

- [x] **LAYOUT-01**: Operator finds three buckets — `READY/`, `SPLIT/`, `AUDIT/` — created directly under `output_root` on every run.
- [x] **LAYOUT-02**: Within each bucket, per-spec artifacts are nested as `<bucket>/<vendor>/<spec>/`, preserving vendor and spec identity.
- [x] **LAYOUT-03**: The per-run `run-<timestamp>-<stem>/` folder is gone; re-running a spec overwrites its prior output in place at `<bucket>/<vendor>/<spec>/`.

### Output Routing (ROUTE)

- [x] **ROUTE-01**: All nine per-spec `main.py` outputs land in `SPLIT/<vendor>/<spec>/` — `cleaned_spec.xlsx`, `classification.jsonl`, `<stem>_annotated.xlsx`, `rows_raw.json`, `rows_normalized.json`, `run_summary.json`, `unknown_rows.csv`, `header_rows.csv`, `run.log`.
- [x] **ROUTE-02**: The `branded` workbook lands in `READY/<vendor>/<spec>/` renamed `Коммерческое предложение_<spec>.xlsx` (filename only; bytes/content unchanged).
- [x] **ROUTE-03**: `batch_audit.py` reads `<stem>_annotated.xlsx` from `SPLIT/<vendor>/<spec>/` and writes `<stem>_annotated_audited.xlsx` to `AUDIT/<vendor>/<spec>/`.
- [x] **ROUTE-04**: Batch-level aggregates — `audit_report.json`, `audit_summary.xlsx` (`batch_audit.py`), `cluster_summary.xlsx` (`cluster_audit.py`) — are written to the `AUDIT/` root (no vendor/spec nesting).
- [x] **ROUTE-05**: The TOTAL copy mechanism (`run_manager.copy_to_total` and its call site) is removed; no artifact is duplicated into a TOTAL folder.

### Output Manifest (MANIFEST)

- [x] **MANIFEST-01**: A `README.md` manifest is written at `output_root` describing each artifact as a table of file → bucket → purpose.

### Test Realignment (TEST)

- [x] **TEST-01**: Path/layout tests are updated to assert the new `<bucket>/<vendor>/<spec>/` structure and filenames; golden-fixture **content** stays byte-equal (no `--update-golden`); the pytest suite passes within the skip-ratio gate.

## Out of Scope

Explicitly excluded for v1.2. Documented to prevent scope creep.

| Feature | Reason |
|---------|--------|
| Any change to artifact **content** (columns, translation, formatting, new docs) | v1.2 is routing-only; content work is the next milestone |
| Changes to classification / normalization / audit **logic** | Only destination paths change in `main.py`, `run_manager.py`, `batch_audit.py`, `cluster_audit.py` |
| Regenerating golden fixtures (`--update-golden`) | Goldens stay byte-equal; only path/layout tests are realigned |
| Renaming or re-bucketing any artifact other than `branded` | Single content-adjacent exception is the `branded` → Russian filename rename |
| Cross-platform launcher, CI/pre-commit, taxonomy work | Carried on the v2.x backlog, unrelated to output routing |
