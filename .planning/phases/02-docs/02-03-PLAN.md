The Quick Start sequence the reader will follow:

1. Clone the repo + ensure Python 3.10+.
2. (Optional default) create a venv at `C:\venv` OR use any venv they already have.
3. `pip install -r spec_classifier/requirements.txt` from inside the venv.
4. (Optional) copy `spec_classifier/config.local.yaml.example` → `spec_classifier/config.local.yaml`; edit `paths.input_root` / `paths.output_root` if defaults aren't desired. Defaults: `%USERPROFILE%\Desktop\INPUT`, `%USERPROFILE%\Desktop\OUTPUT`.
5. Place vendor `.xlsx` under `%USERPROFILE%\Desktop\INPUT\<vendor>\`.
6. Run `.\run.ps1` (full pipeline, all six vendors) or `.\run.ps1 -Vendor huawei -NoAi -SkipTests` (smoke test, single vendor, no OpenAI key).

    # Teresa — Excel Spec Classifier

    Deterministic, rule-based pipeline that turns vendor hardware Bills of Materials (Excel `.xlsx`) into structured, audited classification output for **Dell, Cisco CCW, HPE, Lenovo (DCSC), xFusion (FusionServer eDeal), and Huawei (eDeal)**.

    > Excel in → parse → normalize → classify → Excel/JSON/CSV out. No ML; everything is YAML rules + regex. Fully reproducible.

    ## Repository layout

    The repo is two things stacked:

    - **Repo root** — a thin Windows launcher: `run.ps1` (PowerShell orchestrator), `teresa.bat` (double-click → GUI), `teresa_gui.py` (PyQt6 GUI). No Python package lives here.
    - **`spec_classifier/`** — the actual codebase. CLI entry (`main.py`), audits (`batch_audit.py`, `cluster_audit.py`), source tree (`src/`), per-vendor YAML rules (`rules/`), pytest suite (`tests/`), frozen golden fixtures (`golden/`), and long-form docs (`docs/`).

    The repo holds **only code**. INPUT specs, OUTPUT runs, and the venv all live outside the repo.

    ## Quick Start (Windows)

    Prerequisites: Windows 10+, **Python 3.10+**, PowerShell.

    ```powershell
    # 1. Clone
    git clone <repo-url> teresa
    cd teresa

    # 2. Create or use a venv. Default suggestion: C:\venv
    #    Override path freely — venv location is your choice.
    python -m venv C:\venv
    C:\venv\Scripts\Activate.ps1
    pip install -r spec_classifier\requirements.txt

    # 3. Optional: customise paths
    Copy-Item spec_classifier\config.local.yaml.example spec_classifier\config.local.yaml
    # Edit spec_classifier\config.local.yaml — set paths.input_root / paths.output_root
    # Defaults if you skip this step:
    #   INPUT  = %USERPROFILE%\Desktop\INPUT\<vendor>\
    #   OUTPUT = %USERPROFILE%\Desktop\OUTPUT\

    # 4. Drop your vendor .xlsx files into INPUT\<vendor>\, e.g.
    #    %USERPROFILE%\Desktop\INPUT\huawei\hu1.xlsx

    # 5. Run the pipeline
    .\run.ps1                                  # all 6 vendors + audits + pytest
    .\run.ps1 -Vendor huawei -NoAi -SkipTests  # single-vendor smoke
    ```

    Output appears in `%USERPROFILE%\Desktop\OUTPUT\<vendor>_run\run-YYYY-MM-DD__HH-MM-SS-<stem>\`.

    ## Configuration

    Two-layer YAML overlay:

    - `spec_classifier/config.yaml` — committed defaults (relative paths).
    - `spec_classifier/config.local.yaml` — gitignored, your absolute paths. Copy from `config.local.yaml.example` and edit. Local file wins, deep-merged over committed defaults.

    `C:\venv` is the suggested venv location, not a hard requirement — point your shell at any venv before running `pip install`. Override INPUT / OUTPUT roots in `config.local.yaml`.
