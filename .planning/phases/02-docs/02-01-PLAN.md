The 5 critical business rules (root CLAUDE.md must contain these verbatim per D-01):

1. `power_cord` has `hw_type=None` intentionally. Absent from `device_type_map` in every vendor YAML, with explicit comments. `batch_audit.py:_E8_NO_HW_TYPE_DEVICES = {"power_cord", "enablement_kit"}` excludes it from E8. Do not "fix" the missing mapping. The `power_cord ≈ cable` alias in `batch_audit.DEVICE_TYPE_ALIASES` is **only** for AI-mismatch suppression.
2. LOGISTIC = packaging, documents, freight only. Power cords, stacking cables, rails, brackets are HW, not LOGISTIC.
3. BASE without `device_type` is normal (E15 = INFO). BASE with `device_type` is valid; E6/E10 must not fire. Only `hw_type` on BASE triggers E10.
4. `is_factory_integrated=True` rows are CONFIG; AI does not check them.
5. `hw_type_rules.applies_to` is `[HW]` only — never `[HW, BASE]`.
