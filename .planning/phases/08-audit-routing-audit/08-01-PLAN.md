<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem subpath -> AUDIT write path | The `<vendor>/<spec>` segment of each AUDIT write path is derived via `f.relative_to(SPLIT_root).parent`, where `f` is a path discovered by rglob under the operator-controlled `output_root/SPLIT/`. The subpath originates from Phase 7 output (vendor = trusted VENDOR_REGISTRY key, spec = input file stem). |
| CLI `--output-dir` | Operator-supplied OUTPUT root (local, trusted operator). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Tampering / Information Disclosure | AUDIT write path derivation (`AUDIT_root / f.relative_to(SPLIT_root).parent / ...`) | accept | Path traversal would require an attacker to plant a crafted `<vendor>/<spec>/` subtree (with `..` segments) under the operator's local `output_root/SPLIT/`. `relative_to(SPLIT_root)` keeps the result relative to SPLIT; the mirror is re-rooted under AUDIT, so it cannot escape AUDIT to write elsewhere. The SPLIT subtree is produced by Phase 7's trusted `main.py` (vendor = registry key, spec = input stem) on the same local machine — not a network/remote input. Residual risk: a malicious local actor with write access to OUTPUT already controls the machine. Low value, local-only. |
| T-08-02 | Denial of Service | `out_path.parent.mkdir(parents=True, exist_ok=True)` on derived AUDIT paths | accept | mkdir mirrors an existing, finite SPLIT subtree (one dir per discovered annotated file). No unbounded recursion or attacker-driven fan-out beyond the count of input specs the operator already chose to process. |
| T-08-03 | Tampering | Removal of `{vendor}_run`/`hp_run`/`-TOTAL` matchers | accept | Pure dead-code deletion of pre-Phase-7 layout matchers; narrows, not widens, the path-detection surface. No new input is trusted. |

**No high-severity threats.** All path tokens derive from a trusted local registry + the
Phase-7 output tree on the operator's own machine; there is no network or third-party input
crossing this boundary. `relative_to` + AUDIT re-rooting bounds writes inside AUDIT/. No
blocking (high) threat — nothing to escalate.
</threat_model>
