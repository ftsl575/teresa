    **Important framing (per W-3): Step 4 below is HEURISTIC, not authoritative.** It only catches `from <module>` imports — it does NOT catch `import <module>` (bare imports), `from <parent_pkg> import <child>` (child-name imports), dynamic imports via `importlib`, string-based registry lookups, or test-only imports under `tests/`. Any "POTENTIAL ORPHAN" surfaced by step 4 is a CANDIDATE for human review, not a confirmed orphan. The plan does NOT delete based on step 4 output — it surfaces and gates on Task 3.2 (human checkpoint).

    Commands (PowerShell):
    ```powershell
    # 1) Tracked dead-file check (must be empty)
    git ls-files | Select-String -Pattern '(commits\.txt|\.bak$|\.zip$|\.tmp$)'

    # 2) Untracked dead files at repo root (commits.txt is the known one)
    Test-Path commits.txt
    Get-ChildItem -File -Filter '*.bak' -Recurse -ErrorAction SilentlyContinue
    Get-ChildItem -File -Filter '*.tmp' -Recurse -ErrorAction SilentlyContinue
    Get-ChildItem -File -Filter '*.zip' -Recurse -ErrorAction SilentlyContinue

    # 3) Confirm verify_teresa_audit_actionables.py reference status
    git grep -n 'verify_teresa_audit_actionables' -- 'spec_classifier/**'

    # 4) Source-tree orphan HEURISTIC grep (per W-3: documented as heuristic; surfaces candidates for human review).
    #    Walks every spec_classifier/src/**/*.py and checks for ANY import-shape reference. If zero, surfaces as POTENTIAL ORPHAN.
    #    Patterns checked:
    #      - `from <full.dotted.module>` — fully-qualified `from` imports
    #      - `from <parent_dotted_pkg> import <leaf_name>` — child-name imports
    #      - `import <full.dotted.module>` — bare module imports
    #      - bare leaf-name reference in registries / string keys (catches VENDOR_REGISTRY-style maps)
    #    Patterns NOT checked (this is why the grep is heuristic):
    #      - importlib.import_module(<runtime-string>) — dynamic imports
    #      - getattr() / __import__() — reflection
    #      - YAML / JSON config that names modules as strings (rules engine doesn't do this — confirmed at plan time)
    #    Therefore: ANY POTENTIAL ORPHAN surfaced MUST be human-reviewed in Task 3.2 before deletion (D-09 conservative policy applies).
    git ls-files spec_classifier/src/ | Where-Object { $_ -match '\.py$' -and $_ -notmatch '__init__\.py$' } | ForEach-Object {
        $file = $_
        # Build dotted module path from repo-root path
        $dotted = ($file -replace '^spec_classifier/', '' -replace '\.py$','' -replace '/','.')
        # Build parent package + leaf name for `from <pkg> import <leaf>` detection
        $parts = $dotted -split '\.'
        $leaf = $parts[-1]
        $parent = ($parts | Select-Object -SkipLast 1) -join '.'

        # Search for any import-shape that references this module
        $hits = @()
        $hits += git grep -l "from $dotted" 2>$null
        $hits += git grep -l "import $dotted" 2>$null
        if ($parent) {
            # `from <parent> import <leaf>` — match leaf as a token after `import`
            $hits += git grep -l -E "from $([regex]::Escape($parent)) import (\w+, *)*\b$([regex]::Escape($leaf))\b" 2>$null
        }
        # Also check for bare leaf-name reference (registry / string key) — even more heuristic
        $hits += git grep -l "\b$leaf\b" -- 'spec_classifier/main.py' 'spec_classifier/*.py' 2>$null

        $hits = $hits | Where-Object { $_ -ne $null -and $_ -ne '' } | Sort-Object -Unique
        if ($hits.Count -eq 0) {
            Write-Host "POTENTIAL ORPHAN (heuristic — needs human review): $file" -ForegroundColor Yellow
        }
    }
    ```
