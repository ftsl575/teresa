<objective>
Wire `PYTHONPYCACHEPREFIX` and `PYTEST_ADDOPTS` from `config.local.yaml::temp_root` into `teresa_gui.py` so they are set in the GUI process BEFORE `QApplication` is constructed and BEFORE any `subprocess.Popen` call. The spawned PowerShell child (`launch_run_ps1`) inherits the env vars from the parent process; `run.ps1` (Plan 04-01) also re-sets them redundantly — defense-in-depth per D-13.

Purpose: Closes the runtime-cache architectural gap for the GUI dispatch path (CACHE-02). Without this plan, GUI users who launch `run.ps1` via `subprocess.Popen` from a freshly opened GUI process get cache pollution INSIDE the repo until `run.ps1` itself sets the env vars further downstream — but `subprocess.Popen` resolution of imported modules in the GUI process can already drop `__pycache__` next to `teresa_gui.py` before `run.ps1` ever executes. Setting the env var in `main()` before `QApplication` is the load-bearing fix.

Output: A new `_discover_temp_path()` method on `TeresaWindow` (modeled line-for-line on `_discover_input_path()` at `teresa_gui.py:397-408`) plus a new env-var setting block at the top of `main()` that resolves `temp_root` once and exports both env vars before `QApplication(sys.argv)`.

```python
def main():
    # Redirect runtime cache artifacts (Python bytecode, pytest cache) to temp_root
    # BEFORE QApplication and BEFORE any subprocess.Popen so spawned PowerShell
    # children inherit the env vars. This is defense-in-depth — run.ps1 also sets
    # these from the same temp_root source per D-13.
    cfg = REPO_ROOT / "spec_classifier" / "config.local.yaml"
    temp_root = str(Path.home() / "Desktop" / "temporary")
    if cfg.exists():
        try:
            text = cfg.read_text(encoding="utf-8")
            for line in text.splitlines():
                line = line.strip()
                if line.startswith("temp_root:"):
                    temp_root = line.split(":", 1)[1].strip().strip('"').strip("'")
                    break
        except Exception:
            pass
    os.environ["PYTHONPYCACHEPREFIX"] = str(Path(temp_root) / "__pycache__")
    pytest_cache_arg = f"-o cache_dir={Path(temp_root) / '.pytest_cache'}"
    if os.environ.get("PYTEST_ADDOPTS"):
        os.environ["PYTEST_ADDOPTS"] = f"{os.environ['PYTEST_ADDOPTS']} {pytest_cache_arg}"
    else:
        os.environ["PYTEST_ADDOPTS"] = pytest_cache_arg

    app = QApplication(sys.argv)
    app.setStyle("Fusion")  # consistent base look across Windows versions
    win = TeresaWindow()
    win.show()
    sys.exit(app.exec())
```

The `if os.environ.get("PYTEST_ADDOPTS")` branch implements D-10's append-on-existing semantics (mirrors the partner logic in `run.ps1` from Plan 04-01). The block MUST sit BEFORE `app = QApplication(sys.argv)` per D-13. Imports already in place at the top of `teresa_gui.py` (`os`, `sys`, `Path`) are sufficient — do NOT add any new top-level imports.
